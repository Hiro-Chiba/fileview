## 強みと弱み

### FileView

**強み:**
- ゼロ設定で即座に動作
- ターミナル画像プロトコルを自動検出 (Kitty/iTerm2/Sixel等)
- コンパクトな3.4MBバイナリ (Luaサポート込み)
- 高速起動 (2.3ms)
- 狭いターミナル向けアダプティブステータスバー
- **AI-first設計**: MCPサーバー、Claude Code連携、コンテキスト用ツリー出力

**弱み:**
- 小さなコミュニティ (個人開発)
- 未成熟なプラグインエコシステム
- 部分的な非同期I/O (yaziは完全非同期)
- 限定的なドキュメント

### Yazi

**強み:**
- 完全な非同期I/O (大規模ディレクトリでもUIがブロックしない)
- 成熟したLuaプラグインエコシステムとパッケージマネージャー
- アクティブなコミュニティと豊富なドキュメント
- プラグイン開発用の型定義

**弱み:**
- 最適な体験には設定が必要
- 高いメモリ使用量 (~38MB)
- まだ活発に開発中 (破壊的変更あり)

### nnn

**強み:**
- 極めて軽量 (0.1MBバイナリ、3.4MBメモリ)
- 10年以上の安定性と成熟度
- POSIX準拠、どこでも動作
- 環境変数ベース (設定ファイルなし)

**弱み:**
- 組み込み画像プレビューなし
- カスタムキーバインド (デフォルトでvim風ではない)
- モダンな機能にはプラグインが必要

### lf

**強み:**
- 単一Goバイナリ、依存関係なし
- サーバー/クライアントアーキテクチャ (ターミナル間でコピー可能)
- シェルスクリプトで高度にカスタマイズ可能
- 安定して成熟

**弱み:**
- プレビューには広範なスクリプトが必要
- 組み込み画像プレビューなし
- マウスサポートなし

### ranger

**強み:**
- 最も成熟してドキュメントが充実
- 大きなコミュニティとプラグインエコシステム
- Pythonベース (拡張が容易)

**弱み:**
- 遅い起動 (400ms)
- 高いメモリ使用量 (28MB)
- Python依存

## 正直な評価

FileViewは「バランス型」を目指していますが、以下の間に挟まれています:
- **yazi**: 機能、非同期I/O、エコシステムで優位
- **nnn**: 軽量性と安定性で優位

**独自の差別化要因: AI連携**

FileViewは組み込みAIツーリングサポートを持つ唯一のターミナルファイルマネージャーです:
- **MCPサーバー**: Claude Codeが直接ファイルをブラウズ・読み取り可能
- **Claude向け出力**: `Ctrl+Y` でシンタックスヒント付きコピー
- **ツリー出力**: `--tree` でインタラクションなしにAIコンテキストを取得
- **選択モード**: `--select-mode` でAI駆動ファイル選択

これにより FileView は **AIペアプログラミングワークフローの事実上の選択肢** となります。

FileViewは以下のユーザーに最適です:
1. AIコーディングアシスタント (Claude Code、Cursor等) を使用
2. ドキュメントを読まずに即座に動作するものを求める
3. 狭いターミナル (Claude Code、tmux分割) を使用
4. 最大の機能より簡潔さを重視

## 他ツールからの移行

### rangerから
- 同じvimキーバインド (j/k/h/l)
- 設定なしでプレビューが動作
- Python依存なし
- はるかに高速な起動

### lfから
- 似たキーバインド
- 組み込みプレビュー (スクリプト不要)
- 組み込みファジーファインダー

### yaziから
- よりシンプル、設定不要
- 低いメモリ使用量
- 少ない機能 (意図的)
- より良い狭い画面サポート
