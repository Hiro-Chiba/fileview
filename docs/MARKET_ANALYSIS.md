# FileView 市場分析 & 独自路線戦略

## 競合分析

### 1. Yazi（最大の競合）

**GitHub**: [sxyazi/yazi](https://github.com/sxyazi/yazi)

| 項目 | 内容 |
|------|------|
| 言語 | Rust |
| 強み | 非同期I/O、プラグインシステム（Lua）、タブ、マルチペイン |
| 弱み | 複雑、設定が必要、学習コスト高い |
| ポジション | ヘビーユーザー向け最強ツール |

**Yazi の特徴**:
- Lua プラグインで無限に拡張可能
- zoxide, fzf, ripgrep 連携
- 動画・PDF・アーカイブプレビュー
- タブ、デュアルペイン

---

### 2. nnn（最軽量）

| 項目 | 内容 |
|------|------|
| 言語 | C |
| 強み | 最速、最軽量、ゼロ依存 |
| 弱み | 機能限定的、画像プレビュー弱い、UI シンプルすぎ |
| ポジション | ミニマリスト向け |

---

### 3. lf（バランス型）

| 項目 | 内容 |
|------|------|
| 言語 | Go |
| 強み | シングルバイナリ、ranger 風ワークフロー |
| 弱み | 設定にシェルスクリプト必要 |
| ポジション | ranger から移行したいユーザー |

---

### 4. ft / filetree（直接競合）

**GitHub**: [nyanko3141592/filetree](https://github.com/nyanko3141592/filetree)

| 項目 | 内容 |
|------|------|
| 言語 | Rust |
| 強み | VSCode 風インターフェース、Vim キーバインド |
| 特徴 | Git 状態表示、環境変数でコマンド設定 |

**fileview との類似点**:
- Rust 製
- Vim キーバインド
- Git 状態表示
- マウスサポート
- Nerd Fonts 対応

**⚠️ 差別化が必要**

## 市場ポジションマップ

```
軽量 ←────────────────────────────────────→ 高機能

nnn        lf       fv / ft      ranger       yazi
 │          │           │           │           │
 ▼          ▼           ▼           ▼           ▼
最速      バランス   シンプル    レガシー    最強
ミニマル   ranger風  ゼロ設定    Python     プラグイン
```

**fileview のポジション**: 「シンプルで高速、ゼロ設定」

---

## fileview の現在の強み

| 強み | 競合との比較 |
|------|-------------|
| **画像プレビュー自動検出** | Kitty/iTerm2/Sixel/Chafa/Halfblocks を自動選択。yaziも対応しているが、fileviewの方がシンプル |
| **ゼロ設定** | `cargo install` 直後に使える。yaziは設定推奨、lfは設定必須 |
| **stdin モード** | `find \| fv --stdin --pick` でパイプライン統合。他ツールにはない |
| **シェル連携** | `--pick`, `--on-select`, `--choosedir` の組み合わせ |
| **軽量** | yaziより軽い、nnnより機能豊富 |

## 独自路線戦略

### 基本方針

> **「設定不要で、すぐ使える、見るためのツール」**

yaziと機能で競争しない。シンプルさと「すぐ使える」体験で差別化。

### やること

1. **画像プレビューのさらなる強化**（最大の差別化）
2. **シェル連携の強化**（CLIツールとしての価値）
3. **起動速度の維持**（yaziより軽い）
4. **ゼロ設定の維持**（設定ファイル不要）

### やらないこと

- プラグインシステム（yaziの領域）
- タブ / マルチペイン（yaziの領域）
- 設定ファイル（シンプルさを損なう）

## 改善ロードマップ

### 優先度: 高

| 機能 | 理由 | 工数 |
|------|------|------|
| **アーカイブ内容プレビュー** | zip/tar.gz の中身を表示。yaziにはあるがfileviewにない | 中 |
| **PDF ページナビゲーション** | 現在は1ページ目のみ。ページ送りができると便利 | 中 |
| **起動時間ベンチマーク** | 「yaziより速い」を数値で証明 | 低 |

### 優先度: 中

| 機能 | 理由 | 工数 |
|------|------|------|
| **JSON 出力強化** | `--format json` で構造化出力。スクリプト連携向上 | 低 |
| **動画サムネイル** | ffmpeg 連携で動画の最初のフレームを表示 | 中 |
| **Office 文書プレビュー** | docx/xlsx の内容を表示（外部ツール連携） | 高 |

### 優先度: 低（検討中）

| 機能 | 理由 | 懸念 |
|------|------|------|
| **zoxide 連携** | 頻繁に使うディレクトリへのジャンプ | 依存関係増加 |
| **カスタムプレビューア** | 環境変数で外部プレビューア指定 | 設定の複雑化 |

## ft (filetree) との差別化

ft と fileview は非常に似ている。差別化ポイント:

| 観点 | fileview | ft |
|------|----------|-----|
| 画像プレビュー | 5プロトコル自動検出 | 要確認 |
| stdin モード | あり | なし？ |
| シェル連携 | --pick, --choosedir, --on-select | 環境変数のみ |
| 公開場所 | crates.io | GitHub のみ？ |

**戦略**: fileview はシェル連携とプレビューで勝負。
