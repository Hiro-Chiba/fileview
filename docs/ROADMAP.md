# FileView - Roadmap

## What is FileView?

ターミナル向けのシンプルなファイルビューワー。

```
ls より便利、yazi より軽い
```

## Current Status: v0.11.0

主要機能は実装済み:

- ファイルツリー表示（Vim風キーバインド）
- Git状態表示（色分け、ブランチ名）
- プレビュー（テキスト、画像、ディレクトリ情報、Hex）
- 画像プロトコル自動検出（Kitty, iTerm2, Sixel, Chafa, Halfblocks）
- ファジーファインダー（Ctrl+P）
- ファイル操作（作成、削除、リネーム、コピー、移動）
- シェル連携（--pick, --on-select, --choosedir）
- マウスサポート、ドラッグ&ドロップ

## Design Principles

### やること

1. **シンプルに保つ** - 機能追加より安定性
2. **高速起動** - ターミナルツールは軽くあるべき
3. **ゼロ設定** - インストールしてすぐ使える
4. **画像プレビュー** - これが差別化ポイント

### やらないこと

1. **yaziと競争しない** - 機能数で勝てない、勝つ必要もない
2. **プラグインシステム** - 複雑さの元
3. **設定ファイル** - シンプルさを損なう
4. **AI連携機能** - Claude Codeは既にファイルを読める

## Position

```
軽い ←──────────────────────────────→ 重い

ls    tree    fv    lf    ranger    yazi
              ↑
           ここ
```

fileviewは「ls/treeより便利だけど、yazi/rangerより軽い」ポジション。

- [ ] Critical/Highバグ: 0
- [ ] テストカバレッジ: 80%+
- [ ] ドキュメント完備

## What's NOT on the Roadmap

以下は実装しない:

| 機能 | 理由 |
|------|------|
| AI連携（MCP等） | Claude Codeが既にファイルを読める |
| コンテキストコピー | 同上 |
| プラグインシステム | 複雑さの元 |
| 設定ファイル | ゼロ設定を維持 |
| タブ/分割ウィンドウ | スコープ外 |
| リモートファイル | スコープ外 |
